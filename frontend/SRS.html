Specificația Cerințelor de Sistem

ElectroRecommender – Platformă Web de Recomandare a Dispozitivelor Electronice

Autori: Baciu Leonard-Matei și Marin Vasile Răzvan

Rezumat

Acest document specifică cerințele de sistem pentru ElectroRecommender, o platformă web care recomandă utilizatorilor dispozitive electronice (telefoane, tablete, laptopuri, smartwatch-uri, drone etc.) pe baza caracteristicilor, prețului, recenziilor și a surselor multiple (API-uri, scraping, RSS). Platforma integrează motor de recomandare, management centralizat al produselor și surselor, fluxuri RSS pentru noutăți/statistici și o extensie de browser bonus. Specificația respectă structura IEEE SRS pentru aplicații software web.

1. Introducere

1.1 Scop

Scopul acestui document este de a specifica cerințele platformei ElectroRecommender, un sistem web care centralizează și recomandă dispozitive electronice folosind date din surse variate, cu motor de recomandare inteligent și funcționalități de administrare, filtrare, statistică și export RSS.

1.2 Domeniu de aplicare

Sistemul ElectroRecommender VA oferi:

  • Autentificare utilizatori și roluri (guest, membru, admin)
  • Recomandare dispozitive pe baza prețului, caracteristicilor și preferințelor
  • Integrare cu surse multiple: scraping, API, RSS
  • Management complet al produselor, surselor, știrilor și utilizatorilor
  • Căutare, filtrare, sortare avansată pe criterii diverse
  • Generare de fluxuri RSS pentru noutăți, top produse/statistici
  • UI responsive și extensie de browser pentru recomandări rapide (bonus)

Sistemul NU VA oferi:

  • Procesare plăți sau integrare cu magazine (doar linkuri externe)
  • Funcționalități de chat, forum sau recenzii proprii
  • Notificări push/email

2. Descriere generală

2.1 Perspectiva produsului

ElectroRecommender este o aplicație web multi-layer, ce integrează:

  • Frontend: UI HTML5/CSS3/JS fără framework-uri, design responsive
  • Backend: Node.js + API REST custom (fără framework), logică de recomandare și admin
  • Bază de date: MongoDB NoSQL (produs/surse/știri/utilizatori)
  • Extensie browser (bonus): Recomandări rapide direct din orice site

2.2 Funcționalitățile produsului

  • Autentificare/înregistrare, profile, roluri
  • Recomandare personalizată și generală (home, pagini categorii, detalii)
  • CRUD complet pe produse, surse, știri, utilizatori (panel admin)
  • Interfață căutare, filtrare, sortare avansată
  • Statistici și generare feed RSS dinamic (cele mai populare, noutăți)
  • Import automat din surse externe (API, RSS, scraping)
  • UI complet responsive pentru desktop, tabletă, mobil
  • Extensie browser: afisează lista celor mai noi/top produse pe orice site

2.3 Caracteristici utilizatori

  • Vizitator: Poate vizualiza recomandări, căuta și filtra produse, citi știri și statistici
  • Membru: Salvează preferințe, poate primi recomandări personalizate
  • Admin: Controlează produse, surse, utilizatori, importuri și rapoarte

2.4 Constrângeri

  • Frontend-ul nu folosește framework-uri (doar vanilla JS/HTML/CSS)
  • Backend custom Node.js fără framework web
  • Baza de date MongoDB, persistă totul server-side
  • Respectarea GDPR, prevenire XSS/SQL injection (input validation, output encoding)

2.5 Presupuneri și dependențe

  • Utilizatorii folosesc un browser modern
  • Acces la internet stabil
  • Acces la API-urile și RSS-urile surselor externe

3.2 Cerințe de performanță

  • Timp încărcare pagină < 3 secunde
  • Timp răspuns API < 2 secunde
  • Import automat produse la fiecare 6-12 ore
  • Funcționează corect cu cel puțin 10 utilizatori simultan

3.3 Constrângeri de proiectare

  • Nu se folosesc framework-uri JS (React, Vue etc.)
  • Design complet responsive, valid HTML5/CSS3
  • Separare clară UI, API și logică business
  • Baza de date doar MongoDB (NU SQL)

3.4 Atributele sistemului software

  • Securitate: parolele criptate, JWT, rate-limiting, validare XSS
  • Fiabilitate: backup periodic, error handling detaliat
  • Ușurință utilizare: navigare intuitivă, feedback UI vizibil
  • Mentenenabilitate: cod modular, comentat, cu fișiere separate pe funcționalitate
